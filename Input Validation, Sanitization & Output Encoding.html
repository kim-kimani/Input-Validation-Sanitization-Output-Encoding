HTML encoding

Prevent XSS — special characters become harmless entities

HTML-ENCODED (safe for DOM)
<h1>hello</h1>
RAW INPUT (as plain text — DANGEROUS, rendered as HTML)

hello

Why encoding matters: Raw input inserted as HTML executes scripts and renders tags (see the heading above). Encoded version replaces < with &lt; and > with &gt; so the browser shows tags as plain text.

SQL injection simulation

Concatenated vs. Parameterised queries

VULNERABLE: String concatenation
SELECT * FROM users WHERE name = 'Robert'); DROP TABLE students; --'
SAFE: Parameterised query (placeholder + bound values)
SELECT * FROM users WHERE name = ?
→ Parameters: ["Robert'); DROP TABLE students; --"]
How parameterised queries work: The SQL statement and the data are sent separately to the database. The input is never inserted into the SQL string, so malicious code like ' OR '1'='1 cannot change the query logic.
